HISTORY OF JAVASCRIPT


JavaScript Overview

JavaScript is a high-level, dynamic, and interpreted programming language that has become a cornerstone of web development. Created by Brendan Eich in 1995 while working at Netscape, JavaScript was initially designed for client-side scripting on the web.

Key Features

- Client-side execution: JavaScript runs on the client's web browser, enabling dynamic interactions without server requests. - Dynamic typing: Variables can hold values of any data type, providing flexibility. - First-class functions: Functions can be passed as arguments, returned, and assigned to variables. - Prototype-based inheritance: Objects can inherit properties and behavior from other objects.

Uses

- Front-end web development: JavaScript is used for creating interactive web pages, web applications, and single-page applications (SPAs). - Back-end development: Node.js allows JavaScript to run on the server-side, enabling developers to build scalable and high-performance server-side applications. - Mobile and desktop app development: JavaScript is used in frameworks like React Native and Electron for building cross-platform mobile and desktop applications. - Game development: JavaScript is used in game engines like Phaser for creating browser-based games.

Evolution

- ECMAScript standard: The ECMAScript standard ensures consistency across different browsers and environments. - Regular updates: New features and improvements are introduced regularly, with notable updates including ES6 (ECMAScript 2015), ES7, and beyond.

Advantages

- Ubiquity: JavaScript is supported by most web browsers and devices. - Versatility: JavaScript can be used for both front-end and back-end development. - Large community: JavaScript has a vast and active community, with numerous resources and libraries available.

Challenges

- Browser compatibility: Different browsers may interpret JavaScript code differently. - Security concerns: JavaScript's client-side execution can introduce security risks if not handled properly.

Overall, JavaScript's versatility, ubiquity, and large community make it a fundamental language for web development and beyond.


COMPARE AND CONTRAST THE DIFFERENCE BETWEEN LER, VAR & CONST


Scope

var: Function scope, meaning it's accessible throughout the function it's declared in, including inner blocks. let and const: Block scope, meaning they're only accessible within the block (e.g., if statement, loop, or function) they're declared in.

Hoisting

var: Variables are "hoisted" to the top of their scope, regardless of where the actual declaration is made. This means you can use a var variable before it's declared. let and const: Also hoisted, but not initialized until they're declared. Trying to use them before declaration results in a ReferenceError.

Reassignment

var and let: Can be reassigned. const: Cannot be reassigned. However, if the value is an object or array, its properties or elements can still be modified.

Best Practices

- Use const for values that shouldn't change. - Use let for values that may change. - Avoid using var unless you need function scope.

JavaScript Concepts: Scope, Hoisting, Closure


1. Scope

Scope determines the visibility of variables. JavaScript has global scope, function scope, and block scope (with let and const).


        // Global Scope
        let globalVar = "I'm global";

        function testScope() {
        // Function Scope
        let localVar = "I'm local";
        console.log(globalVar); // ✅ Accessible
        console.log(localVar);  // ✅ Accessible
        }

        testScope();
        // console.log(localVar); ❌ Error: localVar is not defined outside

2. Hoisting

Hoisting means variables and function declarations are moved to the top of their scope before code execution. Only var and function declarations are hoisted.


        console.log(x); // ✅ undefined (because of hoisting)
        var x = 10;

        hoistedFunction(); // ✅ Works
        function hoistedFunction() {
        console.log("I was hoisted!");
        }

        // console.log(y); ❌ Error
        // let y = 5; // let and const are NOT hoisted like var

3. Closure

Closure is when a function remembers and has access to variables from its outer function even after the outer function has returned.


        function outerFunction() {
        let count = 0;

        return function innerFunction() {
            count++;
            console.log("Count:", count);
        };
        }

        const counter = outerFunction();
        counter(); // Count: 1
        counter(); // Count: 2
        counter(); // Count: 3

JavaScript Data Types and Their Methods


1. String Methods


      let text = "Hello, JavaScript!";
      
      console.log(text.length);         // 18
      console.log(text.toUpperCase()); // "HELLO, JAVASCRIPT!"
      console.log(text.toLowerCase()); // "hello, javascript!"
      console.log(text.includes("Java")); // true
      console.log(text.replace("JavaScript", "World")); // "Hello, World!"
      console.log(text.slice(0, 5)); // "Hello"

2. Number Methods


      let num = 42.567;
      
      console.log(num.toFixed(2));   // "42.57"
      console.log(num.toString());   // "42.567"
      console.log(Number.isInteger(num)); // false
      console.log(Math.round(num));  // 43
      console.log(Math.floor(num));  // 42
      console.log(Math.ceil(num));   // 43

3. Boolean Methods (via logic)


      let isTrue = true;
      let isFalse = false;
      
      console.log(Boolean(0));      // false
      console.log(Boolean("Hello"));// true
      console.log(!isTrue);         // false
      console.log(isTrue && isFalse); // false
      console.log(isTrue || isFalse); // true

4. Array Methods


      let fruits = ["apple", "banana", "cherry"];
      
      console.log(fruits.length);        // 3
      console.log(fruits.push("date"));  // Adds "date", returns 4
      console.log(fruits.pop());         // Removes "date"
      console.log(fruits.join(", "));    // "apple, banana, cherry"
      console.log(fruits.indexOf("banana")); // 1
      console.log(fruits.includes("apple")); // true

5. Object Methods


      let person = {
        name: "John",
        age: 30,
        job: "Developer"
      };
      
      console.log(Object.keys(person));   // ["name", "age", "job"]
      console.log(Object.values(person)); // ["John", 30, "Developer"]
      console.log(Object.entries(person));// [["name","John"],["age",30],["job","Developer"]]
      console.log(person.hasOwnProperty("name")); // true

6. Undefined & Null


      let x;
      console.log(x); // undefined
      
      let y = null;
      console.log(y); // null
      
      console.log(typeof x); // "undefined"
      console.log(typeof y); // "object"

7. Function as a Data Type


      function greet(name) {
        return "Hello " + name;
      }
      
      console.log(greet("Alice")); // "Hello Alice"
      console.log(typeof greet);   // "function"
